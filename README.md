# kyusoku

派遣社員のための、ふっと一息つける居場所。

---

## このアプリについて

kyusoku は、派遣社員が日々の気持ちをそっと届けられる日報・ウェルフェアアプリです。

使われなくても、ここにあること自体が「あなたたちのことを気にしている」という意思表明です。解決を約束するのではなく、差し出す手になろうとすることが、このアプリの出発点です。

---

## コンセプト

### 入力する側に、作業を増やさない

3つのアイコン（元気・普通・悲しい）を一押し。それだけで送信できます。書きたい人だけ、書ける。書かない日があっても、それでいい。

### 約束は、嘘をつかない

フォームの送信ボタン下には、こう書かれています。

> この記載内容は担当営業に加工して表示されます

「秘密にします」とは言いません。見られます。でも扱われ方が誠実だと分かること——それが心理的安全性の正体だと考えています。

### 沈黙も、ちゃんと届く

書かなくなったこと、来なくなったこと。それ自体がシグナルです。kyusoku はスタンプの変化だけでなく、提出の途切れやアクセスの消失も静かに拾います。

---

## 機能概要

### 気分スタンプ (必須 / 1日1回)

元気・普通・悲しい の3択。一押しで完了します。

### 日報 (任意 / 1日1回)

| 項目 | 内容 |
|------|------|
| 今日のできごと | 自由記述（上限 500文字） |
| 伝えたいこと | 担当営業へのアナウンス（上限 200文字） |

どちらも書かなくて構いません。スタンプだけで十分です。

### リスク検知

送信された内容は Workers AI で処理されます。

- テキストの危険度を 0〜3 の4段階で判定
- 強い語彙はやわらかく加工して管理画面に届ける
- SOS レベルの投稿は加工せず、緊急性を保ったまま伝える
- **判定は提案です。最終判断は人が行います**

書かれなかったことの検知はルールベース（SQL）で行います。AIは使いません。沈黙を数えるのは、モデルではなくクエリの仕事です。

### 確認フラグ

警告・SOS の投稿には、担当営業による確認フラグが付きます。未確認のまま経過した時間が管理画面に表示されます。

## マスコット

ログイン画面に、名前のない生き物がいます。

全体がパステルカラー。目と口だけがパステルグレー。感情を表現しません。ログインのたびにランダムで「起きてる・寝そう・寝てる」のどれかです。

何も語りかけてきません。励ましません。ただ、そこにいます。

荒んだ日に「そういえば」と思い出して、アプリを開いてもらえたら。それだけでいいと思っています。

---

## データポリシー

- 日報・アクセスログは **1ヶ月で削除**します
- 削除前に月次サマリーに集計して保持します
- サマリーには個人の言葉を残しません。スコアと集計値のみです
- 担当営業は自分の担当者のデータのみ閲覧できます

## 設計の姿勢

kyusoku は完璧なアプリを目指しません。

小さな声を完璧に汲み取ることは、今の段階ではできません。誤検知は起こります。運用が追いつかない日もあります。それでも、差し出した手は本物です。

知見を溜めながら、少しずつ育てていきます。
