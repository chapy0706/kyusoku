# ADR-0002: 危険度判定は原文に対して行う

- 日付: 2026-05-27
- ステータス: 承認済み

---

## コンテキスト

kyusoku は送信されたテキストを2段階で処理する。まず危険度を判定し、次に管理画面向けにマイルド化する。この2つの処理には「どちらを先に行うか」という順序の問題がある。

マイルド化を先に行うと実装が単純になるように見えるが、本質的な問題がある。マイルド化は強い語彙をやわらかく変換する処理であり、その過程で危険度の判断に必要な語彙が失われる可能性がある。つまり、マイルド化後のテキストで判定すると、本来SOSと判定すべき投稿が「正常」と判定されてしまうリスクがある。

## 決定

危険度判定は、かならず原文に対して行う。マイルド化後のテキストで判定してはならない。

処理の順序を以下のように固定する。

```
原文受信 → D1に原文保存 → ユーザーに200返却
  → (waitUntil) 原文に対して危険度判定
  → 原文に対してマイルド化
  → 判定結果・マイルド化後テキストをD1に書き込む
```

## 根拠

- マイルド化は「伝え方」を変える処理であり、「内容の危険度」は変わらない
- 本来拾うべき語彙がマイルド化で消えると、検知の信頼性が根本から失われる
- 原文・マイルド化後テキスト・判定結果を別カラムで持つことで、後から監査や改善が可能になる

## 影響

- D1のスキーマに `content_raw`・`content_soft`・`risk_level` の3カラムが必要になる
- 原文は保持されるため、データの取り扱いには注意が必要（ADR-0005のデータ保持ポリシー参照）
- 管理画面は通常 `content_soft` を表示し、原文確認には明示的な操作を要する
