# ADR-0008: マスコットはデータを持たず、機能を背負わない

- 日付: 2026-05-27
- ステータス: 承認済み

---

## コンテキスト

kyusoku のログイン画面には、名前のない生き物がいる。このマスコットの役割は「場の空気」を担うことであり、機能的な目的を持たせないことが設計方針だ。

しかし開発を進める中で、「気分スタンプと連動させる」「ユーザーの状態を反映させる」「メッセージを持たせる」といった機能追加の誘惑が生まれることが予想される。これらは一見「より親切」に見えるが、マスコットの本質を損なう。

## 決定

マスコットはいかなるデータも持たない。ユーザーの気分スタンプ・リスクレベル・提出履歴と連動させない。

状態はログイン時にランダムで「起きてる・寝そう・寝てる」の3択のみ。このランダム性に意味はない。意味がないからこそ、ユーザーは自由に受け取れる。

## 根拠

- マスコットがユーザーの状態を鏡のように反映すると、「観察されている」感覚を生む
- 「悲しい」を押したマスコットが悲しい顔になると、落ち込みが増幅される可能性がある
- 機能を背負わせないことが、「安心の場所」としての純粋さを守る唯一の方法だ
- ミロのヴィーナス——欠けた部分があるからこそ人は補完したくなる——と同じ原理で、余白があるからこそユーザーはそこに自分を置ける
- 「居ても居なくてもよさそうだけど、なんかいる」という曖昧な存在感が、荒んだ日に「そういえば」と思い出してもらうアンカーになる

## 影響

- マスコットはフロントエンドのみで完結する。Workers・D1・AIとは一切接続しない
- 実装コストは最小。CSS/SVGアニメーションと乱数で十分
- 将来、機能追加の提案が来たときは、このADRを参照して判断する
- 感情を与える・励ます・メッセージを話す、といった変更は、このADRの決定を覆す新ADRが必要
