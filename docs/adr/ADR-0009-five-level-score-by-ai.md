# ADR-0009: 5段階評価はAIが算出し、ユーザーに評価作業をさせない

- 日付: 2026-05-27
- ステータス: 承認済み

---

## コンテキスト

管理側としては、派遣社員の状態を5段階で把握したい。しかし、ユーザーに「今日の状態を5段階で評価してください」と入力を求めることは、このアプリの設計原則「入力する側に作業を増やさない」と直接矛盾する。

信頼が薄れている人たちに評価作業を課せば、「どうせ意味ない」という反応を引き出す可能性が高い。入力の負荷が高いほど、継続率は下がる。

## 決定

5段階評価はユーザーが行わない。AIとルールベースの分析を合算して、管理画面にのみ表示する。

評価の算出に使う入力は以下の通り。

| 入力 | 担当 |
|------|------|
| 気分スタンプの値と変化傾向 | SilenceAnalyzer（ルールベース） |
| 提出率・記述率・アクセス率 | SilenceAnalyzer（ルールベース） |
| テキストの危険度スコア | TextRiskClassifier（Workers AI） |

これらを合算した総合スコアを1〜5に正規化して `RiskAssessment` が出力する。ユーザーの画面には一切表示しない。

## 根拠

- ユーザーに評価させると「評価する作業」が生まれ、継続のハードルになる
- AIとルールで代わりに算出することで、ユーザーはスタンプひとつで参加できる
- 本人が「評価している」という意識を持たずに、自然なデータが集まる方が精度も高い

## 影響

- `RiskAssessment` の正規化ロジックは、運用を通じて調整が必要になる
- 5段階の重み付けは初期値を設定し、知見が溜まったら新ADRで更新していく
- 管理画面にはスコアとその根拠（どのシグナルが高かったか）も合わせて表示することで、担当営業が判断しやすくする
