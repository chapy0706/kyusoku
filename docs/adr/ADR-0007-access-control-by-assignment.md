# ADR-0007: 担当営業は自分の担当者のデータのみ閲覧できる

- 日付: 2026-05-27
- ステータス: 承認済み

---

## コンテキスト

kyusoku は送信フォームに「この記載内容は担当営業に加工して表示されます」と明示する。これは倫理的な約束だ。

この約束を果たすためには、閲覧権限の設計が「担当営業」という言葉と一致していなければならない。もし担当営業が担当外の派遣社員のデータも見られる状態であれば、ユーザーへの約束と実態が食い違う。信頼が薄れている人たちを相手にするアプリで、最初の約束が嘘になることは許容できない。

## 決定

担当営業は自分の担当者のデータのみ閲覧できる。担当外のデータにはアクセスできない。

ロールを以下の3段階に分ける。

| ロール | 閲覧範囲 |
|--------|----------|
| 派遣社員 | 自分の送信履歴のみ |
| 担当営業 | 自分が担当する派遣社員のデータのみ |
| 管理者 | 全ユーザーのデータ |

担当者の紐付けはユーザー登録時に決定する。

## 根拠

- 「担当営業に表示されます」という約束を、権限設計で担保する
- 閲覧範囲の設計は機能要件であると同時に、倫理要件でもある
- 誰がどの範囲を見ているかが明確であれば、将来的に「誰が見たか」の監査も可能になる

## 影響

- ユーザーテーブルに担当営業との紐付けカラムが必要になる
- 担当営業が変わった場合の移行フローを、実装時に決める必要がある
- APIレイヤーで担当者フィルタを必ず適用する。フロントの表示だけで制御してはならない
